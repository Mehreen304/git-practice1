Objective:-
The objective of this lab was to gain practical knowledge of software engineering tools used for project management and version control.For this‚ we created a Trello account and a GitHub account․ Trello was used to manage software engineering projects through the use of boards‚ lists and cards for files and folders‚ and assigning tasks, tracking progress, setting deadlines, and collaborating with team members‚ which correspond to features within the software․ The tutorial also introduced the user to using the distributed version control system Git‚ using the commands for initializing a repository‚ checking status‚ staging files‚ committing changes‚ setting the user and pushing to a remote repository hosted on GitHub․

Outcome:-
As a conclusion of the lab‚ we generated our Trello ID and GitHub ID to use Trello to track our project tasks and GitHub as a source code host․ Also‚ we learned how to use basic Git commands‚ such as git initializing‚ git status‚ git add‚ git commit with a relevant commit message‚ and git config to set configuration values․ We also learned how to use the git push command to push local commits to a remote GitHub repository‚ and how Git keeps project history‚ and allows collaboration and version control․Overall, we became familiar with industry-standard tools for project planning, team collaboration, and source code management used throughout the software development lifecycle.
